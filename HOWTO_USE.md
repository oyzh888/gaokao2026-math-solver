# 如何用它辅助做题

这个项目不只是一次评测——它沉淀了一套**「用大模型 + agentic 工具啃下一道硬数学题」
的可复用工作流**。下面是把它用起来的几种方式。

---

## A. 把它当「多模型陪练」——一道题问多个模型

不同模型在不同步骤上各有所长（见 [BENCHMARK.md §6](BENCHMARK.md)）。遇到一道硬题，
**别只问一个模型**，并行问 3–4 个，再比对：

```bash
for m in gpt-5.5 ; do python runners/run_openai.py    prompts/your_problem.txt $m > out_$m.md; done
python runners/run_anthropic.py prompts/your_problem.txt claude-opus-4-5 > out_opus.md
python runners/run_gemini.py    prompts/your_problem.txt gemini-2.5-pro  > out_gemini.md
```

**怎么读结果**：
- **结论一致** 的小问 → 可信度高。
- **结论分歧** 的小问 → 正是难点所在，重点人工复核（本题的 (3ii) 就是典型）。
- 把多份解答喂回给一个强模型，让它**交叉审稿**：「这几份解答对第 X 问的证明，哪份
  最严格？哪里有跳步？」往往比任何单份都好。

---

## B. 难步骤上，开 harness（让模型写代码自验证）

本项目最强的结论：**最难的一步，纯推理会卡，但允许模型写代码做数值实验、再据此
归纳证明，就能突破**（Opus harness 用 200 万随机点逆推出「禁带引理」）。

实操（在 [Claude Code](https://claude.com/claude-code) 或任何能跑代码的 agent 里）：

1. 给出题面，要求：**「先写 Python 做数值实验摸清结构 / 找反例，再据此写严格证明」**。
2. 让它把猜想（如「某区间上 f 有上界」）**先用随机采样验证**，失败的猜想立刻丢弃。
3. 让它把每一步推理**用代码 double-check**（如本题 `model_runs/scripts/` 里的脚本：
   构造满足条件的反例函数、暴力搜违反包含关系的位移）。
4. 数值站得住了，再要求它**把数值洞察翻译成纸面证明**。

> 适用场景：抽象函数、不等式、组合构造、数论——凡是「能编程验证猜想」的题，
> harness 的增益最大。纯几何/纯符号变形题增益较小。

---

## C. 把它当「题库脚手架」——加你自己的题

仓库已经把流程模板化，加一道新题只要：

1. 把**干净题面**（⚠️ 不要带答案/提示）放进 `prompts/your_problem.txt`。
2. 用 `runners/` 跑你想测的模型，输出存到 `solutions/<problem>/model_runs/<model>.md`。
3. 在 `scripts/build_compare.py` 的 `MODELS` 列表里加条目 + 你的人工评分。
4. `python scripts/build_compare.py` 重新生成对比页。

---

## D. 出题人 / 老师视角：用它探「这题区分度在哪」

把一道题喂给一排模型，**模型集体卡住的小问 = 真正的区分点**。本题里 (1)(2)(3i)
几乎人人会、(3ii) 才拉开差距——这和考场数据（T19 平均完成度 <30%）吻合。
可以用这个信号快速判断一道新题/模拟题的难度梯度是否合理。

---

## ⚠️ 用的时候记住

- **模型会自信地说错**：老模型连本题 (1) 都算错却写得理直气壮。**结论必须人工核**，
  尤其是证明题的「最后一步」和「显然成立」。
- **别把提示喂进去**：想测真实能力，prompt 里就只放题面（见本项目纪律）。想要它
  帮你解题、不在乎评测公平，那随便给提示无妨——两个目的别混。
- **单次采样有随机性**：重要结论多跑几次 / 多问几个模型再下定论。
